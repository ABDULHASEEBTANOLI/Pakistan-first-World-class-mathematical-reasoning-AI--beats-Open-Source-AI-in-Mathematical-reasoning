⚡
Swap = Is Project Ki Backbone

32 GiB RAM + 150 GB SSD swap = ~182 GB effective address space. mergekit ka Task Arithmetic merge 3×32B models ke saath 60-80 GB peak pe jata hai — swap ke bina OOM error aata hai. DeepSpeed ZeRO-3 bhi NVMe offloading use karta hai. Yeh step miss karna = project fail karna.

📋 /etc/fstab + sysctl · Azure VM SSH Session Mein
# ── STEP 1: Check existing swap ──────────────────────────────
free -h
swapon --show

# ── STEP 2: 150 GB swap file banao (SSD pe — /mnt is SSD) ───
sudo fallocate -l 150G /mnt/swapfile
sudo chmod 600 /mnt/swapfile
sudo mkswap /mnt/swapfile
sudo swapon /mnt/swapfile

# ── STEP 3: Permanent karo (reboot ke baad bhi rahe) ─────────
echo '/mnt/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# ── STEP 4: Swappiness optimize karo ─────────────────────────
# 10 = RAM prefer karo, sirf zaroorat pe swap. Training ke liye 60 better.
echo 'vm.swappiness=60' | sudo tee -a /etc/sysctl.conf
echo 'vm.vfs_cache_pressure=50' | sudo tee -a /etc/sysctl.conf
sudo sysctl -p

# ── STEP 5: Verify karo ────────────────────────────────────────
free -h
# Expected output:
#               total        used        free
# Mem:            32G         ...         ...
# Swap:          150G           0         150G  ← YAHAN DIKHNA CHAHIYE

Wrong property values (e.g., yield strength, melting point) engineering failures cause kar sakte hain. Solution: question aane pe pehle Materials Project API query karo, verified data context mein include karo, phir HAIDER se answer lo. Hallucination dramatically reduce hota hai.

from mp_api.client import MPRester
import re

def haider_rag_prompt(question: str, mp_key: str) -> str:
    """Build RAG-augmented prompt from Materials Project data."""
    context = ""
    
    # Try to extract material formula from question
    formula_match = re.search(
        r'\b([A-Z][a-z]?\d*)+\b', question  # simple formula pattern
    )
    
    if formula_match:
        formula = formula_match.group()
        try:
            with MPRester(mp_key) as mpr:
                docs = mpr.materials.summary.search(
                    formula=formula,
                    fields=["material_id", "formula_pretty",
                            "band_gap", "formation_energy_per_atom",
                            "density", "crystal_system"],
                    num_chunks=1
                )
                if docs:
                    r = docs[0]
                    context = f"""
=== Verified Data: Materials Project ({r.material_id}) ===
Formula: {r.formula_pretty}
Band Gap: {r.band_gap:.3f} eV
Formation Energy: {r.formation_energy_per_atom:.3f} eV/atom
Crystal System: {r.crystal_system}
Density: {r.density:.3f} g/cm³
========================================"""
        except:
            context = "[Materials Project query failed — answering from training knowledge]"
    
    prompt = f"""You are HAIDER — an expert in Materials Science, Metallurgy, and Materials Engineering.
You adapt your style to the user. Show step-by-step reasoning.
When unsure of precise values, explicitly state uncertainty.

{context if context else "[No database match — use training knowledge + acknowledge uncertainty if needed]"}

### Question:
{question}

### Answer:"""
    
    return prompt

# Example usage
if __name__ == "__main__":

    print(prompt)
